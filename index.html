Plaits.js

Introduction 

Plaits is a form creation, validation and rendering library for Node.js.

It supports both synchronous and asynchronous validation (via Promises), and comes with many built-in validators which are provided by the excellent validator.js library.

It is intended for use with Express, but can be used with other libraries and frameworks too.

Source is available on GitHub, and comes with a large suite of unit tests.

Form Models 

Model Creation 

Plaits models are simply Backbone models with different functionality.

A model must be given a name, and a list of fields. It also takes an optional list of validators (explained later).

A basic model example:

// Require Plaits
var Plaits = require('plaits');

// Register Form Declaration
var RegisterForm = Plaits.Model.extend(
  {
    // Form Name
    name: 'register_form',
    // Field List
    fields: [
      'username',
      'email_address',
      'password'
    ],
    // Validators
    validators: {
      username: [
        Plaits.Validators.required(),
        Plaits.Validators.alphanumeric()
      ],
      email_address: [
        Plaits.Validators.required(),
        Plaits.Validators.email()
      ],
      password: [
        Plaits.Validators.required(),
        Plaits.Validators.minLength(8)
      ]
    }
  }
);

// Export Form
module.exports = RegisterForm;

Models can also take a labels object as part of their declaration;

labels: {
  'username': 'Your Username'
}

Model Instantiation 

To instantiate a form model, simply require & use new:

// Require
var RegisterForm = require('../forms/register');

// New Form
var registerForm = new RegisterForm();

Provided Validators 

Plaits comes with a large set of built in validators which should be able to handle most of your use cases.

All built in validators take an optional custom error message if you want to override the default error message, and many will take certain parameters to define their behaviour, for example minLength will take a parameter to define what the minimum length should be.

Required 

required(customErrorMessage)

Whether this field is required.

In most cases, this will check whether the value is null or not - null values are considered invalid.

When generating labels using the Html helpers and templates, the label text will be appended with an asterisk.

Email 

email(customErrorMessage)

Checks if the provided value is an email address.

Minimum Length 

minLength(minLength, customErrorMessage)

Checks that the value matches or exceeds the minimum length provided.

Maximum Length 

maxLength(maxLength, customErrorMessage)

Checks that the value matches or is less than the maximum length provided.

Form Request Parsing 

If you are using the Express framework, you can have the module intelligently parse requests made to your routes:

// Register Form
var RegisterForm = require('../forms/register');

// Test Route - GET
router.get('/test', function (req, res) {
  // New Form
  var registerForm = new RegisterForm();
  // Render, Passing Form
  res.render('test', {
      registerForm: registerForm
  });
});

// Test Route - POST
router.post('/test', function(req, res) {
    // New Form
    var registerForm = new RegisterForm();
});
